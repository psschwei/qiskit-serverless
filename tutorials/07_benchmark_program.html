In order to test compute resources we need to have benchmark program.

Below you can see executon graph of a program. Each of boxes is a task in a program. This program is designed to tests vertical and horizontal scalability of compute resources.

Program accepts list of arguments to control stress levels: - n_entries - number of circuits and observables will be used in each iteration on generation task. - depth_of_recursion - defines how many recursive iterations generation task will have. (Number of generated circuits will be n_entries * depth_of_recursion). - n_backends - number of backends, which defines how many parallel trainspilation and estimations will happen. - n_qubits - number of qubits will be used in a program. Affects backends, circuits and observables. - circuit_depth - depth of generated circuits. - size_of_observable - size of generated observables. - n_graphs - number of times graph above will be executed within one program.
